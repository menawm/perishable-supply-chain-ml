Banana Quality Prediction

This project develops machine learning models to classify banana quality in supermarket supply chains. The goal is to reduce operational losses by identifying low-quality bananas early using features such as sweetness, acidity, and ripeness. The code covers data preparation, model training, and model evaluation.

Step 1: Load and Clean the Data

Load dataset, standardize target column (quality), and convert to factor for classification

banana <- read.csv("banana_quality.csv")
names(banana)[names(banana) == "Quality"] <- "quality"
banana$quality <- as.factor(banana$quality)

Convert quality into a factor (classification labels: good or bad)

banana$quality <- as.factor(banana$quality)

Step 3: Split the Data

Split data into training (80%) and test (20%) sets to assess performance on unseen observations.

# Set seed for reproducibility (so we always get the same split)
set.seed(123)
# Split the data: 80% for training, 20% for testing
train_index <- createDataPartition(banana$quality, p = 0.8, list = FALSE)
train_data <- banana[train_index, ]
test_data <- banana[-train_index, ]

Train and Test Data

Train and evaluate Logistic Regression, KNN, and Decision Tree models on the same data for comparison.

Model 2: K-Nearest Neighbors (KNN)

Scale features and apply KNN to classify banana quality based on nearest neighbors.

# Remove the 'quality' column before scaling the numeric features
train_x <- train_data[, -which(names(train_data) == "quality")]
test_x <- test_data[, -which(names(test_data) == "quality")]

# Scale the training data
train_x_scaled <- scale(train_x)

# Scale the test data using the same parameters as the training set
test_x_scaled <- scale(
  test_x,
  center = attr(train_x_scaled, "scaled:center"),
  scale = attr(train_x_scaled, "scaled:scale")
)

# Extract the labels (target variable)
train_y <- train_data$quality
test_y <- test_data$quality

# Run KNN with k = 5
knn_preds <- knn(train = train_x_scaled, test = test_x_scaled, cl = train_y, k = 5)
# Evaluate performance using confusion matrix
confusionMatrix(knn_preds, test_y)

Model 3: Decision Tree

Train a decision tree to classify banana quality and evaluate an interpretable, rule-based model.

# Train the decision tree model

tree_model <- rpart(quality ~ ., data = train_data, method = "class")

# Make predictions on the test set
tree_preds <- predict(tree_model, test_data, type = "class")
rpart.plot(
  tree_model,
  type = 2,
  extra = 104,
  fallen.leaves = TRUE,
  tweak = 1.2
)

The tree uses feature-based splits such as sweetness, ripeness, and harvest time to classify quality through clear, interpretable rules.

Interpretation:

The decision tree model performs well in identifying both good and bad bananas and provides the added benefit of being easy to interpret. The model’s decision rules can be visualized and explained to non-technical stakeholders.

While it does not outperform KNN in accuracy, it may be preferred in situations where model transparency and explainability are important.

Model Summary: Key Takeaways

All models perform well with distinct strengths:

  • Logistic Regression: Simple, fast baseline with lower accuracy.
  • KNN: Highest accuracy and best for minimizing prediction errors.
  • Decision Tree: Balanced performance with strong interpretability.

Overall, KNN is best for accuracy, the decision tree is best for interpretability, and logistic regression is best for speed and simplicity.
